# AG01 · Agent Runtime：状态、计划、记忆与恢复

- **建议时长：**6 周 · 每周 12-15 小时
- **先修课程：**AP02、AP03
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

可靠 Agent 的核心不是无限自主，而是清楚的状态、终止条件、工具边界、观察轨迹和人工接管。先自己实现循环，再评估框架是否必要。

## 学习目标

- 区分工作流、路由器、Agent 和多 Agent 的适用边界
- 实现状态机、计划、记忆、工具执行、终止与恢复
- 用轨迹而非只看最终答案诊断失败

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 工作流与 Agent；observe-decide-act 循环 |
| 2 | 状态、事件、检查点、终止和预算 |
| 3 | 计划、分解、反思及其失败模式 |
| 4 | 短期状态、长期记忆、检索和遗忘 |
| 5 | 重试、补偿、人工介入和并发 |
| 6 | 自研 Runtime 与框架对照评测 |

## 从零构建

不使用 Agent 框架，实现可插拔模型和工具、显式状态、预算、检查点、终止、恢复及轨迹日志的运行时。

## 真实交付

用该运行时解决一个多步骤真实任务，再与一个成熟框架或固定工作流比较成功率、成本和可维护性。

## 必交证据

- 运行时状态图与接口说明
- 正常、边界和故障轨迹集
- Agent 与固定流程的选型结论

## 反 bypass 口试

1. 什么条件下工作流优于 Agent？
2. 记忆写入错误为何比没有记忆更危险？
3. 如何证明 Agent 真的终止而不是恰好停止？
