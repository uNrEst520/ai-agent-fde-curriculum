# B23 · Build Your Own 推荐与预测实验室

- **对应课程：**[XRS](../curriculum/electives/recommender-time-series.md)
- **建议投入：**50-80 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

一个包含推荐召回/排序与时间序列滚动回测的决策实验室。

## 第一遍约束

先实现简单基线和离线模拟，不把模型分数直接当作业务价值。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 行为与时间数据 | 定义曝光、点击、反馈、时间和目标，检查偏差。 | 区分没看见与看见但没选择。 |
| 2. 推荐基线 | 实现热门、相似度和协同过滤。 | 冷启动和长尾单独报告。 |
| 3. 排序模拟 | 构建召回、特征、排序和多目标权重。 | 离线增益对业务目标可解释。 |
| 4. 时序基线 | 实现 naive、移动平均、季节性和特征回归。 | 不随机打乱时间。 |
| 5. 滚动回测 | 实现 expanding/rolling window 与多步预测。 | 检查未来信息泄漏。 |
| 6. 反馈与线上指标 | 模拟曝光改变数据和长期行为。 | 提出实验、护栏和监控方案。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
