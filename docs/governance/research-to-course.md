# 从论文雷达到正式课程

## 五道门

### 1. 自动发现

每周脚本根据关键词和学科分类抓取候选论文。这个阶段追求不漏掉重要方向，不做质量背书。

### 2. 人工初筛

检查问题是否与 Agent/FDE 能力相关，来源是否为原始论文，摘要结论是否被证据支持。

输出：一张论文候选卡。

### 3. 深度阅读

阅读方法、实验、限制和附录，区分作者结论与课程维护者推断。

输出：完整论文笔记和三个可证伪问题。

### 4. 最小复现

选择一个与课程项目相关的结论进行对照实验。记录模型、提示、数据、成本、结果和失败。

输出：可重复实验与结果记录。

### 5. 教学转化

论文只有在能转化为以下至少一项时才进入正式课程：

- 稳定概念或新的能力目标
- 可操作的设计原则
- 一个反例或故障案例
- 一套新的评测方法
- 一个经过验证的工程实验

## 收录位置

| 证据状态 | 课程位置 |
| --- | --- |
| 只有摘要相关 | 自动雷达 |
| 通过初筛 | 观察列表 |
| 深读但未复现 | 推荐阅读 |
| 已复现、影响特定方向 | 方向专修 |
| 多次验证且跨工具稳定 | 稳定主线候选 |

## 防止课程追热点

- 一篇论文不能单独改变稳定主线。
- Leaderboard 提升不等于真实交付能力提升。
- 新框架必须与简单基线比较。
- 任何“更自主”的 Agent 都要同时评估错误放大、权限和人工接管。
- 课程必须保留失败结果，而不仅是成功 Demo。
