# SE01 · Python 与计算思维

- **建议时长：**6 周 · 每周 12-15 小时
- **先修课程：**O00
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

Python 是 AI 工程最常见的胶水语言。重点不是背语法，而是学会把模糊任务拆成数据、状态、函数、边界和测试。

## 学习目标

- 阅读并追踪中小型 Python 程序
- 使用数据结构、函数、模块、异常和类型表达问题
- 独立定位输入、状态和边界条件导致的错误

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 变量、表达式、输入输出；实现单位转换器 |
| 2 | 分支、循环与不变量；实现规则引擎 |
| 3 | 列表、字典、集合与文件；实现日志分析器 |
| 4 | 函数、模块、类型与异常；拆分可测试结构 |
| 5 | 类、迭代器与标准库；建模一个任务队列 |
| 6 | 测试、调试、性能观察与综合项目 |

## 从零构建

[进入逐步构建指南](../../builds/cli-data-app.md)

只使用标准库构建一个命令行数据应用，并亲手写一个最小断言测试器。

## 真实交付

把应用升级为有配置、日志、错误提示、自动测试和使用说明的可交付工具。

## 主要资源

- [CS50's Introduction to Programming with Python](https://cs50.harvard.edu/python/)
- [Python 官方教程](https://docs.python.org/3/tutorial/)
- [pytest 文档](https://docs.pytest.org/)

资源用于建立主线，最新论文与工具变化从 [研究雷达](../../../research/README.md) 进入候选，不会未经验证直接替换稳定原理。

## 必交证据

- 至少 30 个自动测试
- 三类失败输入和修复记录
- 闭卷重写一个核心模块

## 反 bypass 口试

1. 列表、集合和字典的选择会如何影响行为与性能？
2. 异常应在哪里捕获，为什么？
3. 如何证明一个重构没有改变外部行为？
