# XDS · 领域 AI 工作室

- **建议时长：**8 周 · 每周 12-15 小时
- **先修课程：**EV01、FD01
- **课程性质：**兴趣选修
- **验收方式：**从零构建 + 方向项目 + 口试

## 方向目标

把领域知识、用户研究、AI 能力和生产约束组合成可交付产品。可选择体育、建筑、咨询、教育、制造或其他熟悉领域。

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 领域地图、用户、任务和现有替代方案 |
| 2 | 高价值问题、证据、基线和不做清单 |
| 3 | 人机流程、系统边界和数据计划 |
| 4 | 最小产品和领域专家评审 |
| 5 | 模型/规则/检索/工具的组合实验 |
| 6 | 评测、安全、隐私和失败恢复 |
| 7 | 试点、培训、采用和运营 |
| 8 | 价值复盘、商业提案与脱敏作品集 |

## 从零构建

从空白业务问题建立领域概念模型、规则基线、数据样本和一个最小 AI 能力，不从现成界面倒推需求。

## 真实交付

交付可运行领域产品、用户证据、评测集、风险登记、运行手册、ROI 假设和下一阶段商业化方案。

## 必交证据

- 可运行代码与可复现实验环境
- 基线、切片指标、失败样本和风险记录
- 面向用户或业务负责人的演示与价值说明

## 反 bypass 口试

1. 领域专家知识应进入规则、数据、提示还是评测？
2. 用户喜欢 Demo 为什么仍不代表问题值得解决？
3. 哪些领域判断不应由通用模型最终决定？
