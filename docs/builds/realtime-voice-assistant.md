# B20 · Build Your Own 实时语音助手

- **对应课程：**[XSP](../curriculum/electives/speech-audio.md)
- **建议投入：**55-85 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

一条从麦克风、端点检测、语音识别、任务处理到语音合成，可打断和可评测的实时链路。

## 第一遍约束

先观察波形、频谱和流式缓冲，不把语音输入当作普通文本框。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 音频基础 | 实现采样、分帧、窗函数、频谱和录音回放。 | 能解释采样率与混叠。 |
| 2. 语音活动 | 实现能量基线或 VAD，划分开始、结束和静音。 | 噪声下误触发可测。 |
| 3. 流式识别 | 接 ASR，处理 partial/final、时间戳和错误。 | 首字与最终延迟分别记录。 |
| 4. 对话任务 | 管理轮次、上下文、确认、取消和敏感动作。 | 识别错误不会直接执行高风险动作。 |
| 5. 合成与打断 | 接 TTS，支持播放、barge-in 和恢复。 | 用户说话能停止旧输出。 |
| 6. 端到端评测 | 按噪声、口音、设备、网络和任务切片。 | 同时报告字错率、任务率和延迟。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
