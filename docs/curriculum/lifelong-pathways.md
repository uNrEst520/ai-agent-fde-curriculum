# 长期进阶路线

## 为什么 12 周之后还要继续

12 周解决的是“进入工程和业务交付状态”，无法替代生产经验、系统性计算机基础、领域积累和研究判断。长期能力通过专修、研究和真实交付循环形成。

## 进阶模块

### Agent 可靠性与评测

先修：共同核心。

重点：任务分解、轨迹评测、工具错误、长程退化、恢复策略、可观测性、红队测试和线上回归。

毕业证据：一个包含正常、边界、对抗和故障注入案例的 Agent 评测系统。

### 企业 RAG 与知识工程

重点：数据接入、解析、切分、检索、重排、引用、权限、更新、知识冲突和答案可信度。

毕业证据：一个有来源追踪、权限隔离和离线评测的企业知识应用。

### MCP 与工具生态安全

重点：协议生命周期、资源、提示、工具、传输、授权、用户同意、最小权限和恶意工具防护。

毕业证据：一个自建 MCP 服务、一个客户端接入和一份威胁模型。

### AI Coding 与全栈交付

重点：代码库理解、任务拆解、测试驱动、代码审查、CI/CD、部署、监控和事故复盘。

毕业证据：一个由 AI 辅助开发但本人能够解释、维护和运营的完整产品。

### 多模态与计算机操作

重点：视觉、语音、视频、桌面操作、环境反馈和多模态评测。

毕业证据：一个在真实软件或物理场景中完成任务，并有可重复评测的 Agent。

### FDE 与企业采用

重点：现场调研、流程建模、价值排序、数据和系统接入、试点设计、用户培训、合规、采用和 ROI。

毕业证据：一份企业级 AI 上岗方案和一次真实或仿真部署。

## 推荐的年度循环

```text
第一季度：完成一个技术专修
第二季度：做一次领域项目和用户验证
第三季度：完成一个研究复现实验
第四季度：完成FDE综合交付与能力复评
```

循环不是固定学制。已经上岗的学习者可以直接把工作项目作为交付载体，但必须保留可公开或脱敏的证据。

## 补充计算机基础

当项目暴露出知识缺口时，按需进入长期基础模块：

- 数据结构与算法
- 数据库与事务
- 操作系统与进程
- 计算机网络
- 软件设计与架构
- 安全、隐私与专业伦理
- 概率统计和机器学习基础

这些内容不要求在求职前全部学完，但 L3 以上必须逐步补齐与所负责系统相关的基础。
