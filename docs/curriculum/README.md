# 长期 AI 课程设计

## 培养目标

这套课程覆盖四种互相连接的能力：理解计算系统、理解 AI 模型、构建可靠 AI 应用、把系统部署进真实业务。Agent 与 FDE 是重要出口，但不是课程的全部。

完整课程表直接显示在 [仓库首页](../../README.md#完整课程地图)，所有课程元数据记录在 [catalog.json](catalog.json)。

课程覆盖范围见 [AI 全领域覆盖图](ai-field-map.md)，31 个逐步构建项目见 [Build Your Own AI](../builds/README.md)。

## 为什么需要约两年

三个月可以形成第一个 AI 应用闭环，但不足以同时掌握软件工程、数学、机器学习、深度学习、语言模型、评测安全和企业交付。主线课程的建议学习量约 108 周：

| 学院 | 建议周数 | 核心问题 |
| --- | ---: | --- |
| 入学与学习系统 | 2 | 如何使用 AI 而不绕过理解 |
| 软件与数据工程 | 28 | 软件、数据和网络如何可靠运行 |
| AI 与模型原理 | 35 | 智能系统为什么有效、何时失效 |
| AI 应用与 Agent | 25 | 如何连接模型、知识、工具和用户 |
| 生产系统与 FDE | 18 | 如何上线、治理、采用并产生价值 |

12 门选修方向各需 5-10 周，按目标选择，不要求全部完成。课程允许在满足先修关系后并行学习，因此日历时间不等于所有周数简单相加。

## Build to Understand：两遍构建法

每门课都有两种项目：

1. **理解项目：**减少框架依赖，从零实现一个最小版本，暴露数据结构、算法和系统边界。
2. **交付项目：**使用成熟库和 AI 编程工具，在可靠性、体验、成本和业务约束下完成真实任务。

例如，RAG 课程先实现切分、倒排检索、相似度和引用，再使用向量数据库构建具备权限、更新和评测的知识系统。亲手实现不是为了重复造轮子，而是为了知道轮子为什么转、何时会坏。

## 先修规则

- `prerequisites` 是硬先修。不能解释和完成先修课程的关键项目，不建议进入后续课程。
- 同一阶段内无依赖的课程可以并行，例如 AI 数学与经典 AI。
- 有经验的学习者可以参加挑战测试跳过课程，但仍需提交该课程的构建项目或等价证据。
- 兴趣选修可以穿插在主线中，不必等到毕业项目之后。

## 每门课程的统一结构

- 课程价值与先修诊断
- 可解释、可构建、可评测、可交付四层目标
- 按周学习安排和主要原始资料
- 从零构建项目及里程碑
- 成熟工具对照实验
- 故障、安全与边界案例
- 交付物、评分标准和口试题
- 与后续课程的依赖关系

详情页模板见 [course-unit-template.md](course-unit-template.md)。每门课程必须链接一份独立构建指南，课程与构建项目一一对应并由自动检查保证。

## 四条毕业路线

- [AI Builder / FDE](pathways/ai-builder-fde.md)
- [模型与算法工程](pathways/model-engineering.md)
- [多模态产品](pathways/multimodal-products.md)
- [研究工程](pathways/research-engineering.md)

所有路线共享工程基础、机器学习基础与评测责任。路线只决定学习顺序和选修重点，不降低验收标准。

## 毕业要求

1. 完成主线课程或通过等价挑战测试。
2. 完成至少一门兴趣选修。
3. 完成一次从零构建的模型或系统项目。
4. 完成一次面向真实或高度仿真用户的 FDE 交付。
5. 提交评测集、风险登记、运行手册、业务价值复盘和可脱敏展示材料。
6. 通过 [能力等级模型](competency-framework.md) 的 L3 评审。
