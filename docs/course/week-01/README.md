# Week 01：工程地图与反 bypass 启动

## 本周定位

第一周不是编程速成，也不是让你把 Python、JavaScript、FastAPI 全部学会。本周只训练一件事：**把一个已经跑起来的 AI 产品看成一张工程地图**。

你要从“Codex 帮我做了一个功能”升级为：

```text
我知道用户在哪里输入
我知道前端把什么数据发给后端
我知道后端在哪里处理
我知道结果如何返回页面
我知道测试如何证明功能没有坏
我知道怎么把这件事讲给老板、客户或面试官
```

这正是 FDE 式 AI 业务交付者的第一层能力：进入现场后，先看懂系统、流程、用户和验收标准，再谈 Agent、RAG、MCP 和自动化。

## 本周学习目标

完成本周后，你应该能独立讲清：

- 一个 Web AI 产品的基本组成：前端、后端、API、数据、测试、部署/启动脚本。
- PowerShell 命令的大致结构，而不是死记每个命令。
- Git/GitHub 的最小概念：仓库、文件变更、提交、分支、README。
- 前端页面如何通过 `fetch` 请求后端 API。
- FastAPI 后端如何暴露接口并返回 JSON。
- 测试为什么能减少 bypass，帮助你验收 AI 写的代码。
- 如何用 5 分钟讲清一个已有项目的业务链路和工程链路。

## 每天时间安排

工作日每天 2-3 小时，建议按这个比例：

```text
30 分钟：看教程或文档
45 分钟：读项目里的指定文件
45 分钟：运行命令或观察页面
30 分钟：完成作业记录
15 分钟：整理“我能解释什么 / 我还不懂什么”
```

周末 Day 6-7 用来做小改动、跑验证、写复盘。

## 第一周最终作业

你需要提交一份 `Week 1 工程地图复盘`，内容包括：

1. 项目解决什么业务问题。
2. 用户从打开页面到拿到报告的完整流程。
3. 前端、后端、分析逻辑、报告逻辑、测试分别在哪里。
4. 你本周亲自运行过哪些命令，结果是什么。
5. 你让 Codex 做了哪个小改动，你如何验收。
6. 如果把这个项目带进一家网球培训机构，AI 要真正“上岗”还缺什么。

## 本周最低验收线

你不需要会独立写完整代码，但必须做到：

- 能打开项目并说明启动命令每一段大概什么意思。
- 能找到前端入口、后端入口、报告生成文件和测试文件。
- 能解释 `训练处方` 为什么不只是文案，而是一个早期 Agent 决策层。
- 能把“我看不懂”具体化为某个文件、某个函数、某个概念，而不是一团雾。
