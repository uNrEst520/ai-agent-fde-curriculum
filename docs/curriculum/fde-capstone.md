# FDE 毕业项目：让 AI 真正上岗

## 目标

毕业项目不是做一个聊天页面，而是证明学习者能够把 AI 接入真实业务，使它从“展示能力”进入“承担工作”。

## 必交成果

### 1. 业务诊断

- 目标用户和工作现场
- 当前流程、时间、成本和质量问题
- 为什么值得使用 AI
- 明确不使用 AI 的环节

### 2. 人机协作设计

- 人、模型、规则、数据和外部工具的分工
- 正常流程、失败流程和人工接管
- 权限、隐私和审计边界

### 3. 可运行系统

- 前端或可操作入口
- 后端、数据和模型调用
- 测试、部署和运行说明
- 日志、监控和成本记录

### 4. 评测与风险

- 真实代表性任务集
- 质量、成功率、延迟和成本指标
- 边界、对抗和故障注入案例
- 已知限制和停止条件

### 5. 采用与价值

- 用户培训和操作说明
- 试点计划
- 使用反馈和流程变化
- ROI 或可观察的价值代理指标

### 6. 交付答辩

- 3 分钟说明业务价值
- 5 分钟完成主流程演示
- 10 分钟解释架构、评测和风险
- 15 分钟现场追问和变更

## 推荐课题

- 网球训练机构的动作分析和训练处方系统
- 建筑策划概念到策划文本的企业工作流
- 咨询团队的研究、证据和报告交付 Agent
- 中小企业销售线索整理和跟进 Agent
- 组织内部知识问答与流程执行 Agent

## 不通过条件

- 只有模型对话，没有业务流程。
- 只有成功演示，没有评测和失败处理。
- 无法解释关键数据和工具权限。
- 用虚构指标代替真实观察。
- 项目完全由 AI 生成，学习者无法定位和维护。
