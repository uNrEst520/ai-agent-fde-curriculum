# DL01 · 深度学习与自动微分

- **建议时长：**7 周 · 每周 12-15 小时
- **先修课程：**ML01
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

深度学习把表示学习、反向传播与大规模计算结合起来。本课从计算图和梯度开始，而不是从调用训练器开始。

## 学习目标

- 解释计算图、反向传播、初始化和优化器
- 实现自动微分、神经网络层和训练循环
- 诊断过拟合、梯度异常、数据问题与训练不稳定

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 神经元、张量、计算图和前向传播 |
| 2 | 链式法则、反向传播与标量自动微分 |
| 3 | MLP、激活、初始化和批处理 |
| 4 | 优化器、归一化、正则化与训练稳定性 |
| 5 | 卷积、序列表示和注意力预备 |
| 6 | PyTorch 张量、模块、数据加载和 GPU |
| 7 | 自实现框架与 PyTorch 对照项目 |

## 从零构建

从零实现标量自动微分引擎、MLP、损失函数、SGD/Adam 和训练循环，并通过数值梯度检查。

## 真实交付

使用 PyTorch 训练一个小型分类模型，提交实验追踪、误差分析、资源消耗和模型卡。

## 主要资源

- [Dive into Deep Learning](https://d2l.ai/)
- [PyTorch Tutorials](https://pytorch.org/tutorials/)
- [micrograd](https://github.com/karpathy/micrograd)

资源用于建立主线，最新论文与工具变化从 [研究雷达](../../../research/README.md) 进入候选，不会未经验证直接替换稳定原理。

## 必交证据

- 自动微分单元测试和梯度检查
- 训练失败到修复的曲线
- 自实现与 PyTorch 的差异说明

## 反 bypass 口试

1. 反向传播保存了哪些中间信息？
2. 梯度消失可能来自哪些环节？
3. 训练损失下降为什么不保证产品更好？
