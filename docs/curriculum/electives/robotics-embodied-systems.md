# XRO · 机器人与具身 AI 系统

- **建议时长：**8 周 · 每周 12-15 小时
- **先修课程：**SE04、DL01、XRL
- **课程性质：**兴趣选修
- **验收方式：**仿真构建 + 系统项目 + 安全口试

## 方向目标

把感知、状态估计、定位、规划、控制和学习连成物理世界闭环，理解仿真成功与真实部署之间的系统差距。

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 坐标系、运动学、传感器与执行器 |
| 2 | 噪声、滤波、状态估计和 Kalman filter |
| 3 | 定位、地图和 SLAM 概览 |
| 4 | 路径搜索、运动规划和避障 |
| 5 | PID、轨迹跟踪和控制稳定性 |
| 6 | 视觉感知、抓取与行为策略 |
| 7 | 模仿学习、视觉语言动作模型与 sim-to-real |
| 8 | 仿真集成、安全边界和故障演练 |

## 从零构建

[进入逐步构建指南](../../builds/robot-simulator.md)

在二维仿真中实现带噪声传感、状态估计、A* 规划、PID 控制和碰撞检测的移动机器人闭环。

## 真实交付

完成一个仿真机器人任务，报告成功率、碰撞、能耗、延迟、分布变化和紧急停止策略；真实硬件为可选扩展。

## 主要资源

- [MIT Underactuated Robotics](https://underactuated.csail.mit.edu/)
- [Modern Robotics](https://modernrobotics.northwestern.edu/)
- [ROS 2 Documentation](https://docs.ros.org/en/rolling/)

## 必交证据

- 可重复仿真、状态估计和控制曲线
- 规划失败、传感器故障和紧急停止实验
- 成功率、碰撞、延迟与安全边界报告

## 反 bypass 口试

1. 定位误差如何传导到规划和控制？
2. 仿真成功为什么不能直接证明真实安全？
3. 学习策略与经典控制应如何分工？
