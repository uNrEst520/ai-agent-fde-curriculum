# 研究工程路线

## 适合目标

研究工程师、应用研究员、开源模型工程师，以及需要持续判断论文价值并把成果转成产品能力的学习者。

## 推荐顺序

```text
O00 → SE01 → SE03 + SE04 + DE01
→ AI01 + AI02 → ML01 → DL01 → LM01
→ XRE → 选择一个方向选修
→ XLM（语言模型方向）
→ EV01 → PR01
```

研究雷达贯穿全程，但在 ML01 之前以阅读概念和实验结构为主，不急于追逐最新榜单。

## 作品集要求

- 一份主张、证据、基线、消融和限制完整的论文卡
- 一个环境、数据、代码、种子和结果可复现的实验包
- 原论文结果、复现结果和失败原因的对照
- 一份“是否进入产品或课程”的工程决策报告

## 竞争力

不只会复述论文，而是能够检查证据、复现实验、识别适用边界，并把值得采用的研究转成可维护工程。
