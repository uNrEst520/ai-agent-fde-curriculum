# FD02 · 企业 AI 上岗毕业项目

- **建议时长：**8 周 · 每周 15-20 小时
- **先修课程：**PR01、FD01
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

毕业项目把模型、工程、评测和组织采用放进同一个约束环境。验收对象不是 Demo，而是一个有人使用、可运行、可退出、能解释价值的系统。

## 学习目标

- 负责从发现、方案、构建到部署采用的完整生命周期
- 在数据、权限、安全、成本和组织限制下做取舍
- 用运行证据和业务结果而非功能数量证明价值

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 现场发现、问题基线与利益相关者 |
| 2 | 方案、目标流程、架构、风险和试点章程 |
| 3 | 数据与系统接入；评测集和验收基线 |
| 4 | 最小产品构建与内部评审 |
| 5 | 安全、权限、故障、负载和回滚测试 |
| 6 | 试点部署、用户培训与支持 |
| 7 | 采用、质量、成本和业务指标复盘 |
| 8 | 改进、移交、作品集脱敏与答辩 |

## 从零构建

从空目录建立项目的需求、架构、评测、运行和治理骨架，说明每个部分如何约束最终系统。

## 真实交付

完成真实或高度仿真的 AI 上岗交付包：系统、评测集、风险登记、部署、运行手册、培训、验收、ROI 和移交。

## 必交证据

- 至少一轮真实用户反馈
- 上线前后对照指标
- 故障演练、回滚和移交记录

## 反 bypass 口试

1. 哪个证据最能证明系统产生了业务价值？
2. 试点成功但用户不用，问题可能在哪里？
3. 项目结束后谁对模型、数据和风险负责？
