# B08 · Build Your Own 微型机器学习库

- **对应课程：**[ML01](../curriculum/modules/08-machine-learning-from-scratch.md)
- **建议投入：**55-80 小时
- **难度：**中级
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

仅用 NumPy 实现训练、预测、评测和模型选择的经典机器学习工具箱。

## 第一遍约束

每个算法通过正确性测试后才能与 scikit-learn 对照。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 数据与评测 | 实现切分、标准化、混淆矩阵、precision/recall、ROC-AUC。 | 防止测试集进入训练和调参。 |
| 2. 线性回归 | 实现闭式解、梯度下降、正则化和残差分析。 | 与合成真值和成熟库一致。 |
| 3. 逻辑回归 | 实现 sigmoid、交叉熵、梯度和概率校准。 | 检查极端数值与不平衡数据。 |
| 4. 决策树 | 实现分裂、纯度、停止、预测和剪枝。 | 过拟合与深度关系可视化。 |
| 5. 聚类与 kNN | 实现距离、kNN、k-means 和初始化。 | 不同尺度和异常值下行为可解释。 |
| 6. 模型选择 | 实现交叉验证、基线、切片和实验表。 | 完成自实现与成熟库的差异报告。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
