# B16 · Build Your Own 生产模型网关

- **对应课程：**[PR01](../curriculum/modules/16-production-ai-llmops.md)
- **建议投入：**55-85 小时
- **难度：**高级
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

一个处理认证、路由、配额、缓存、追踪、成本、降级和部署的模型网关。

## 第一遍约束

网关只集中横切能力，不吞并业务逻辑；任何缓存必须考虑隐私和时效。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 统一模型接口 | 定义请求、流事件、usage、错误和 provider adapter。 | fake provider 可做确定性测试。 |
| 2. 身份与配额 | 实现租户、API key 占位、速率和预算。 | 不同租户数据与额度隔离。 |
| 3. 可靠性 | 实现 deadline、有限重试、熔断、fallback 和幂等。 | 供应商故障不会级联。 |
| 4. 路由缓存 | 按任务、质量、成本路由；设计安全 cache key。 | 敏感上下文不会跨用户命中。 |
| 5. 可观测性 | 记录 trace、延迟、token、成本、错误和模型版本。 | 一次用户请求可追踪全链路。 |
| 6. 负载与部署 | 容器化、健康检查、压测、SLO、灰度和回滚。 | 提交容量与事故演练报告。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
