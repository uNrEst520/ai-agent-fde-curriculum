# 精选学习资源

这个目录解决的不是“资料够不够多”，而是“学习者下一步究竟点开哪一页”。

## 正式资源与候选资源

- 正式资源写入 [catalog.json](catalog.json)，必须已经分配到具体课程、具体章节、预计时间和作业。
- 候选资源写入 [教育资源观察库](../../research/education-source-watchlist.md)，可以很优秀，但未经过试学、能力匹配和时间评估前不进入主线。
- 搜索结果页、泛化的 YouTube 关键词和“某某 Awesome List”不能作为正式课程入口。

## 一条资源进入主线前必须回答

1. 它具体训练哪一个能力？
2. 学习者要看哪一章，哪些内容明确跳过？
3. 完成需要多少时间？
4. 有什么练习或输出能证明学会？
5. 为什么它比同类候选更适合当前学习者？
6. 是否能稳定访问，是否必须付费？
7. 最后一次人工打开并核验是什么时候？

任一问题没有答案，资源只能留在候选库。

## 精选原则

每个知识单元默认只保留：

- 1 条主资源：负责系统讲解与练习。
- 0-2 条补充资源：只用于弥补主资源的明确缺口。
- 1 个迁移作业：把外部知识迁移到本仓库的真实项目。

“更多链接”不是质量指标。重复讲同一内容的第二门完整课程通常会被删除。

## 检索渠道

优先从以下渠道发现候选，再回到原始页面核验：

| 渠道 | 适合发现 | 使用限制 |
| --- | --- | --- |
| 大学公开课与官方课程站 | 稳定原理、课程作业、讲义 | 仍需选到具体 lecture/module，不能只贴首页 |
| GitHub 官方组织与作者仓库 | 代码、notebook、练习、更新记录 | stars 只代表关注度，不代表教学匹配 |
| Coursera、edX 与 Class Central | 发现结构化课程 | 聚合页只用于发现；访问和收费以课程官方页为准 |
| 官方文档与协议规范 | API、框架、标准的准确用法 | 不适合从头通读，必须指定章节 |
| 论文、作者代码与课程实验 | 新理论与研究复现 | 先进入研究雷达，不直接替换稳定主课 |

## 资源失效或收费时怎么办

1. 先在 issue 中标记失效日期、地区和现象。
2. 保留原资源记录，不悄悄替换。
3. 从候选库选择训练同一能力、作业强度相近的替代项。
4. 由维护者实际完成关键章节和一个作业。
5. 更新 catalog 的 URL、取舍说明和 last_verified，再进入正式课程。
