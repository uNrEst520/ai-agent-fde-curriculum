# PR01 · 生产 AI、LLMOps 与可观测性

- **建议时长：**6 周 · 每周 12-15 小时
- **先修课程：**EV01、DE01
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

能在本机演示的系统距离生产可用仍有延迟、成本、并发、可观测性、发布、回滚和数据治理等大量工作。

## 学习目标

- 设计模型网关、缓存、队列、限流、降级和多模型路由
- 追踪请求、模型、提示、工具、成本与用户反馈
- 建立部署、回归、监控、事故响应和持续改进流程

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 生产架构、SLO、容量、延迟和成本预算 |
| 2 | 模型网关、路由、缓存、批处理和限流 |
| 3 | 日志、指标、trace、反馈和数据闭环 |
| 4 | 容器、配置、密钥、CI/CD 与基础设施 |
| 5 | 发布、灰度、回滚、漂移和事故响应 |
| 6 | 压力测试、安全检查与生产演练 |

## 从零构建

实现带统一接口、配额、重试、缓存、用量统计和 trace id 的最小模型网关。

## 真实交付

将一个 AI 应用部署到可重复环境，完成负载测试、仪表板、告警、运行手册和故障演练。

## 主要资源

- [Full Stack Deep Learning](https://fullstackdeeplearning.com/course/2022/)
- [OpenTelemetry 文档](https://opentelemetry.io/docs/)
- [Docker 官方文档](https://docs.docker.com/)

资源用于建立主线，最新论文与工具变化从 [研究雷达](../../../research/README.md) 进入候选，不会未经验证直接替换稳定原理。

## 必交证据

- SLO、成本和容量预算
- 一次灰度或回滚演练
- 从用户问题追踪到模型/工具调用的 trace

## 反 bypass 口试

1. 缓存为什么可能造成安全或时效问题？
2. 高可用和正确性冲突时如何决策？
3. 哪些日志不应该记录？
