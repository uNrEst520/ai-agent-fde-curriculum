# B21 · Build Your Own 微型扩散模型

- **对应课程：**[XGM](../curriculum/electives/generative-media-diffusion.md)
- **建议投入：**70-110 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

从前向加噪、去噪网络、训练目标到条件采样和可重复工作流的生成模型。

## 第一遍约束

先在小型灰度或低分辨率数据上完成扩散核心，不直接从复杂工作流界面开始。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 数据与噪声 | 准备小数据，定义时间步和前向加噪。 | 不同时间步分布可视化。 |
| 2. 去噪网络 | 构建小型卷积/UNet 预测噪声或目标。 | 形状、时间编码和梯度测试通过。 |
| 3. 训练目标 | 实现随机时间步、损失、EMA 可选和 checkpoint。 | 固定种子可比较训练变化。 |
| 4. 反向采样 | 实现逐步去噪并保存中间状态。 | 从纯噪声得到非随机结构。 |
| 5. 条件控制 | 加入类别/文本简化条件与 guidance。 | 控制强度与多样性有对照。 |
| 6. 工作流评测 | 记录模型、种子、提示、素材来源和失败。 | 质量、符合度、多样性、安全分开评。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
