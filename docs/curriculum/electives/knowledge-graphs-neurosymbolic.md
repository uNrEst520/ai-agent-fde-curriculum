# XKG · 知识图谱与神经符号 AI

- **建议时长：**6 周 · 每周 12-15 小时
- **先修课程：**AI02、DE01
- **课程性质：**兴趣选修
- **验收方式：**从零构建 + 知识系统项目 + 口试

## 方向目标

学习实体、关系、本体、规则、图查询和图表示，理解结构化知识如何与检索、语言模型和可解释推理结合。

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 实体、关系、三元组、图与身份解析 |
| 2 | RDF、schema、本体和约束 |
| 3 | 图遍历、模式查询、规则和推理 |
| 4 | 图 embedding、链路预测和图神经网络概览 |
| 5 | Knowledge Graph RAG、实体链接和证据路径 |
| 6 | 一致性、更新、权限、评测和综合项目 |

## 从零构建

[进入逐步构建指南](../../builds/knowledge-graph-engine.md)

使用基础数据结构实现三元组存储、模式查询、规则推理和来源追踪，再加入实体链接和证据路径输出。

## 真实交付

构建一个领域知识图谱应用，比较纯文本 RAG、图查询与混合方案在正确性、解释性、更新成本和权限上的差异。

## 主要资源

- [Stanford CS520: Knowledge Graphs](https://web.stanford.edu/class/cs520/)
- [W3C RDF 1.1 Concepts](https://www.w3.org/TR/rdf11-concepts/)
- [Neo4j GraphAcademy](https://graphacademy.neo4j.com/)

## 必交证据

- 可查询三元组库、schema 与规则测试
- 事实、派生关系和答案的来源追踪
- 文本 RAG、图查询与混合方案对照

## 反 bypass 口试

1. 文本切分与实体关系建模分别保留什么信息？
2. 本体约束和数据库 schema 有何不同？
3. 图中一条可达路径为什么未必是有效证据？
