# XRS · 推荐系统与时间序列

- **建议时长：**6 周 · 每周 12-15 小时
- **先修课程：**ML01、DE01
- **课程性质：**兴趣选修
- **验收方式：**从零构建 + 方向项目 + 口试

## 方向目标

覆盖个性化系统与时序预测两类常见业务 AI，强调离线指标、反馈循环、冷启动、回测和决策价值。

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 用户-物品数据、隐式反馈、基线和偏差 |
| 2 | 协同过滤、矩阵分解和 embedding |
| 3 | 召回、排序、多目标与探索 |
| 4 | 时间序列分解、特征、预测与不确定性 |
| 5 | 回测、漂移、泄漏、反馈循环和线上指标 |
| 6 | 业务系统、实验和完整评测 |

## 从零构建

[进入逐步构建指南](../../builds/recommender-forecast-lab.md)

从零实现相似度推荐、矩阵分解和滚动时间序列回测器，并构建线上指标模拟器。

## 真实交付

交付一个推荐或预测系统，明确行为目标、反事实限制、冷启动、业务约束和监控方案。

## 主要资源

- [TensorFlow Recommenders](https://www.tensorflow.org/recommenders)
- [Forecasting: Principles and Practice](https://otexts.com/fpp3/)
- [RecSys Papers](https://recsys.acm.org/recsys-papers/)

最新论文只从 [研究雷达](../../../research/README.md) 进入候选，经最小复现和课程 RFC 后才成为正式必修内容。

## 必交证据

- 可运行代码与可复现实验环境
- 基线、切片指标、失败样本和风险记录
- 面向用户或业务负责人的演示与价值说明

## 反 bypass 口试

1. 离线推荐指标提升为何可能伤害长期用户价值？
2. 时间序列随机切分为何通常错误？
3. 反馈循环会如何改变训练数据？
