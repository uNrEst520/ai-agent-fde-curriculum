# B27 · Build Your Own 因果与实验实验室

- **对应课程：**[XCA](../curriculum/electives/causal-inference-experimentation.md)
- **建议投入：**45-70 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

一个用合成与真实数据展示混杂、实验、加权和准实验假设的因果分析工具箱。

## 第一遍约束

每个估计必须写识别假设；不得把预测模型特征重要性当作因果效应。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 反事实模拟 | 生成处理、结果、混杂和真实效应。 | 能直接比较估计与真值。 |
| 2. 随机实验 | 实现随机化、均值差、标准误和置信区间。 | A/A 测试与功效模拟通过。 |
| 3. 因果图 | 画 DAG，识别后门、碰撞点和调整集。 | 错误控制变量会产生可见偏差。 |
| 4. 倾向加权 | 实现 propensity、IPW、重叠与权重诊断。 | 极端权重有处理策略。 |
| 5. 准实验 | 实现 DiD 或 RDD，并检查前趋势/带宽等假设。 | 安慰剂测试被记录。 |
| 6. 决策报告 | 报告效应、区间、异质性、假设和外推边界。 | 明确哪些结论不能下。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
