# B09 · Build Your Own 自动微分与神经网络引擎

- **对应课程：**[DL01](../curriculum/modules/09-deep-learning-autograd.md)
- **建议投入：**60-90 小时
- **难度：**中级
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

从标量计算图开始，构建反向传播、神经网络层、优化器和训练循环。

## 第一遍约束

自动微分核心不得调用 PyTorch、TensorFlow 或 JAX。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. Value 节点 | 保存值、父节点、操作和梯度，支持基本运算。 | 前向结果与手算一致。 |
| 2. 拓扑反向 | 构建拓扑序并按链式法则累积梯度。 | 共享节点和分支图梯度正确。 |
| 3. 梯度检查 | 用有限差分检查每个算子。 | 相对误差低于自定阈值。 |
| 4. 神经网络模块 | 实现 parameter、neuron、layer、MLP 和激活。 | 参数可枚举、清零和更新。 |
| 5. 损失与优化 | 实现 MSE、交叉熵、SGD、动量和 Adam。 | 训练曲线能解释学习率问题。 |
| 6. 框架对照 | 在同一数据上与 PyTorch 比正确性和速度。 | 提交差异、限制和失败案例。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
