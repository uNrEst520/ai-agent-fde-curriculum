# B24 · Build Your Own 语言模型训练与对齐实验

- **对应课程：**[XLM](../curriculum/electives/lm-training-alignment.md)
- **建议投入：**100-160 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

在预算受限环境中完成数据清洗、基础训练、指令微调或偏好优化、评测和性能剖析。

## 第一遍约束

先在 CPU/小 GPU 上验证正确性和预算，再扩大；数据许可与来源必须记录。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 数据管道 | 采集、许可检查、清洗、去重、过滤和混合。 | 抽样审查与数据卡完成。 |
| 2. 训练基线 | 选小模型/自研模型，建立 loss、任务和成本基线。 | 随机种子与 checkpoint 可复现。 |
| 3. 性能剖析 | 测计算、显存、I/O、吞吐和瓶颈。 | 优化前后正确性不变。 |
| 4. 指令微调 | 构造指令数据、mask、SFT 和验证。 | 与基础模型按任务切片比较。 |
| 5. 偏好优化 | 实现或使用 DPO 等方法，检查数据与目标。 | 不只报告奖励模型分数。 |
| 6. 对齐评测 | 测试帮助性、安全、过拒答、偏差和能力回归。 | 提交预算、收益、风险和停止条件。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
