# B19 · Build Your Own 视觉与视频分析流水线

- **对应课程：**[XCV](../curriculum/electives/computer-vision-multimodal.md)
- **建议投入：**70-110 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

从像素与卷积实验到检测/姿态、时序分析、多模态解释和失败画廊的视觉系统。

## 第一遍约束

先实现基础图像算子和小型分类器，再接预训练模型；不得只展示几个成功视频。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 图像与标注 | 读取像素、颜色、resize、增强和标注格式。 | 可视化确认图像与标签对齐。 |
| 2. 卷积基线 | 从零实现卷积/边缘算子和小型分类器。 | 与框架算子数值对照。 |
| 3. 检测或姿态 | 接入成熟模型，统一 frame、box、keypoint 和置信度。 | 遮挡、多人和漏检可记录。 |
| 4. 时序与跟踪 | 实现身份关联、平滑、事件窗口和动作特征。 | 不把相邻视频帧泄漏到测试集。 |
| 5. 多模态解释 | 把视觉证据转成结构化结论，再生成自然语言。 | 语言主张可回指帧和关键点。 |
| 6. 场景评测 | 按设备、光线、角度、人群和动作切片。 | 提交速度、准确、失败画廊和接管。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
