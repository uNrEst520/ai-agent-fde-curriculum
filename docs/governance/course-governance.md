# 课程治理与版本规则

## 治理目标

AI 课程必须同时解决两个相反问题：

- 不能因为模型和工具快速变化而过时。
- 不能因为追逐每个新工具而失去稳定知识主干。

因此课程采用分层治理。

## 四类内容

### 稳定核心

进入条件最高。内容应跨模型、框架和平台成立，例如 API、状态、工具边界、评测、权限和业务交付。

复审周期：每年。除非存在明确错误，不在学习者进行中途大幅改动。

### 方向专修

围绕稳定职业能力组织，可以随工程实践更新。

复审周期：每季度。

### 工具实验

用于学习当前模型、SDK、平台和工作方式，不承诺长期保留。

复审周期：每月。必须标注版本和检查日期。

### 研究观察

新论文、基准和协议首先进入这里，不直接成为必修。

复审周期：每周自动发现，每月人工评审。

## 变更流程

1. 提交问题：失效、重复、缺失、证据变化或学习效果问题。
2. 写 RFC：问题、证据、方案、替代方案和迁移影响。
3. 完成最小教学实验或项目实验。
4. 由至少一名维护者和一名学习者评审。
5. 合并并更新变更记录。
6. 对重大变化提供迁移说明。

## 版本含义

- Major：能力目标、先修关系或毕业标准改变。
- Minor：课程或项目替换，但能力目标保持。
- Patch：链接、环境、文字和顺序修复。

## 退出机制

内容出现以下情况时应移出主路径：

- 原始资源长期无法访问。
- 工具已经停止维护或存在明显安全风险。
- 与其他单元重复且没有额外学习价值。
- 没有作业或评测可以证明学习效果。
- 结论被更可靠证据推翻。

移出的优质内容可以进入历史、扩展阅读或研究档案，不需要永久删除。
