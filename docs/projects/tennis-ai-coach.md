# 随身网球教练 MVP

## 一句话

一个手机优先的网球动作分析 Web Demo：上传或录制正手、反手、发球、截击视频，后端提取人体骨架，生成关键帧、骨骼叠加视频、中文动作报告和 AI 教练训练处方。

## 解决的问题

业余球员训练时常见的问题不是没有内容，而是“不知道当前动作最该先改哪里”。这个项目把视频证据转成可执行建议，让球员每次只关注 1-3 个最容易落地的调整点。

## 核心功能

- 支持正手、反手、发球、截击四类动作。
- 支持视频上传、浏览器录制、iPhone H.265 转码。
- 使用 MediaPipe Pose 提取人体关键点和动作阶段指标。
- 生成关键帧、骨骼叠加视频、中文动作指导报告。
- 支持参考动作库、标注、指纹、素材包、样本包和 Markdown 报告导出。
- 新增 `training_prescription`：把动作分析结果转成“诊断、原因假设、下一组训练、验收标准、Agent 决策链”。

## 技术栈

- Frontend：Vite、React、lucide-react
- Backend：FastAPI、Python
- Vision：MediaPipe Pose，预留 YOLO 球拍检测 / 球拍关键点训练
- Testing：pytest、Playwright smoke scripts、Vite build

## AI Builder 亮点

- 不是单点 AI 调用，而是完整的“上传 -> 分析 -> 报告 -> 处方 -> 复测”闭环。
- 训练处方采用规则 Agent 思路，先保证稳定、可解释、低成本，再为后续 LLM 接入留下接口。
- 数据集导出、人工复核队列和参考动作库说明项目具备持续迭代能力。

## 商业化方向

- 面向网球教练：降低课后复盘成本。
- 面向网球相机/运动硬件公司：作为动作分析和训练建议层。
- 面向个人球友：形成低成本随身教练工具。

## Demo 讲法

打开 Web 页面，上传一段动作视频，等待报告生成。报告页展示总评分、关键建议、训练处方、下一组训练、参考动作匹配和导出包。面试时重点讲清楚：系统如何从视频证据转成可执行训练，而不是只展示一个分数。
