# 长期路线图

## v0.1：课程基础设施

- 建立长期课程地图和能力等级模型。
- 把 12 周定义为共同核心，而不是完整学习终点。
- 建立研究雷达、论文阅读模板和课程治理流程。
- 建立自动检查和 GitHub 协作模板。

## v0.2：完成共同核心

- 补齐 Week 02-12 的逐日课程、作业、记忆清单和评分标准。
- 每周绑定一个真实项目任务和一项可运行验证。
- 建立阶段测验与第一次 FDE 小型交付。

## v0.3：首批方向专修

- Agent 可靠性与评测。
- 企业 RAG 与知识工程。
- MCP、工具生态与权限安全。
- AI Coding 与全栈交付。
- FDE 需求发现、部署采用与价值复盘。

## v0.4：开放课程社区

- 建立学习者作业展示和同伴评审规则。
- 建立课程 RFC、论文讨论和复现实验记录。
- 发布课程版本说明和资源失效报告。

## 长期循环

每季度执行一次：

1. 检查外部课程、官方文档和工具是否失效。
2. 审查研究雷达中高价值论文。
3. 复现至少一个值得进入课程的结论。
4. 更新一个实践模块，不随意改动稳定核心。
5. 发布一次版本说明和学习路径影响分析。
