# B29 · Build Your Own 二维机器人闭环模拟器

- **对应课程：**[XRO](../curriculum/electives/robotics-embodied-systems.md)
- **建议投入：**80-130 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

一个有传感噪声、状态估计、规划、控制、碰撞和安全停止的移动机器人仿真。

## 第一遍约束

真实硬件不是必需；先在可重复仿真中证明闭环、故障和安全。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 世界与运动学 | 定义地图、机器人姿态、速度和时间步。 | 直线、转弯和边界与预期一致。 |
| 2. 传感噪声 | 模拟里程计/距离传感与随机噪声。 | 噪声分布和 seed 可复现。 |
| 3. 状态估计 | 实现互补/Kalman 简化滤波。 | 估计误差相对原始测量改善。 |
| 4. 路径规划 | 实现栅格、A*、膨胀障碍和重规划。 | 无路时安全失败。 |
| 5. 轨迹控制 | 实现 PID 或 pure pursuit，处理饱和。 | 跟踪误差与稳定性可视化。 |
| 6. 故障与安全 | 注入传感失真、障碍突现、延迟和断连。 | 碰撞率、停止距离和恢复报告完成。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
