# AI Agent / FDE 长期课程地图

## 课程定位

这套课程培养的不是某个框架的操作员，而是能够借助 AI 完成业务交付的人。学习者既要理解稳定原理，也要持续追踪技术变化，并在真实项目中证明能力。

## 四层课程体系

### 1. 共同核心

周期：12 周，每周 10-15 小时。

目标：建立工程地图、编程与 Web 基础、LLM 应用原理、Agent 工作循环、RAG/MCP、评测安全和第一次 FDE 交付。

共同核心完成后达到能力等级 L2：能够在 AI 辅助下独立完成一个边界清楚的小型交付。

### 2. 方向专修

周期：每个方向 6-10 周。至少完成一个主修和一个辅修。

- Agent 可靠性、评测与可观测性
- 企业 RAG 与知识工程
- MCP、工具生态与权限安全
- AI Coding 与全栈产品交付
- 多模态、语音、视觉与计算机操作
- 私有部署、成本与性能工程
- FDE 业务发现、组织采用与商业价值

### 3. 前沿研究实验室

持续进行，每月选择一篇论文或一个重要协议变化：

1. 阅读原始来源。
2. 写一张论文证据卡。
3. 完成最小复现或对照实验。
4. 判断对现有项目和课程有什么影响。
5. 必要时提交课程变更 RFC。

### 4. FDE 真实交付

每半年至少完成一次真实或高度仿真的业务交付，从问题发现一直走到部署、培训、验收和价值复盘。

## 学习顺序

```text
入学诊断
→ 12周共同核心
→ 第一次FDE小型交付
→ 选择主修方向
→ 每月研究实验
→ 半年度真实交付
→ 能力复评与下一阶段路线
```

共同核心内部按先修关系推进；方向专修可以并行；研究实验和真实项目贯穿长期学习。

## 学习证据

每个单元都必须产生以下证据中的至少四项：

- 自己的概念解释
- 系统或数据流图
- 对现有代码的定位和追踪
- 可运行的代码改动
- 自动测试或人工评测
- 失败案例与风险记录
- 业务价值说明
- 演示视频或交付文档

## 课程出口

课程没有唯一终点。不同阶段对应不同出口：

| 等级 | 可以承担的工作 |
| --- | --- |
| L1 | 在指导下操作和解释已有 AI 应用 |
| L2 | 借助 AI 工具独立完成边界清楚的小型功能 |
| L3 | 负责一个可上线 AI 应用及其评测、监控和迭代 |
| L4 | 进入企业现场，设计跨系统 Agent 流程并推动采用 |
| L5 | 定义组织级 AI 交付方法、平台和治理标准 |

详细定义见 [能力等级模型](competency-framework.md)。
