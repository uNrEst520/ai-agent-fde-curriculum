# B22 · Build Your Own 强化学习智能体

- **对应课程：**[XRL](../curriculum/electives/reinforcement-learning-embodied.md)
- **建议投入：**70-110 小时
- **难度：**高级选修
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

从自建 Gridworld 到动态规划、Q-learning、DQN、奖励攻击和可重复策略评测。

## 第一遍约束

先实现环境和表格算法，不直接调用高层 RL 训练框架。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 环境接口 | 定义状态、动作、transition、reward、done 和 seed。 | 环境确定性与边界测试通过。 |
| 2. 动态规划 | 实现 value iteration 和 policy iteration。 | 小环境与手算最优策略一致。 |
| 3. Q-learning | 实现 epsilon-greedy、更新和学习曲线。 | 不同 seed 报告均值与方差。 |
| 4. DQN | 加入网络、replay、target network 和稳定技巧。 | 能定位发散与过估计。 |
| 5. 奖励攻击 | 设计捷径、循环刷分或安全违规。 | 奖励高但行为坏的轨迹可复现。 |
| 6. 策略评测 | 评成功、安全、样本效率、鲁棒和泛化。 | 训练环境外的变化单独测试。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
