# B10 · Build Your Own 微型 Transformer 语言模型

- **对应课程：**[LM01](../curriculum/modules/10-transformer-language-model.md)
- **建议投入：**80-120 小时
- **难度：**中高级
- **学习方法：**每一关都完成“构建 → 验证 → 解释”，再进入下一关

## 最终要造什么

从文本数据到 tokenizer、注意力、Transformer、训练、生成和评测的完整微型语言模型。

## 第一遍约束

先亲手实现关键组件；允许用 PyTorch 张量与 autograd，但不调用现成 Transformer 模型。

这些约束只用于暴露原理。通过核心验收后，应与成熟库或框架对照，不建议把教学实现直接用于高风险生产。

## 逐步关卡

| 关卡 | 亲手构建 | 通过检查 |
| --- | --- | --- |
| 1. 数据与 tokenizer | 清洗小语料，实现字符级或 BPE 编解码和数据切片。 | encode/decode 可逆并记录未知情况。 |
| 2. 语言建模基线 | 实现 bigram 模型和负对数似然。 | 训练损失与随机基线可比较。 |
| 3. 因果注意力 | 实现 Q/K/V、缩放点积、mask 和多头。 | 形状、mask 与手算小例通过。 |
| 4. Transformer block | 加入 embedding、位置、残差、归一化和 MLP。 | 前向和梯度无 NaN。 |
| 5. 训练系统 | 实现 batch、优化、checkpoint、随机种子和日志。 | 中断后恢复结果连续。 |
| 6. 生成与评测 | 实现温度、top-k/p、KV cache 可选扩展和任务评测。 | 比较采样策略、成本和失败文本。 |

## 通用验收

- 从空目录或最小脚手架开始，提交每一关的独立 commit。
- 正常、边界、错误和故障案例都有自动测试或可重复实验。
- 能画出输入、状态、处理、输出和失败路径。
- 能闭卷解释一个关键算法或协议，并现场修改一个小行为。
- 记录 AI 提出的关键建议、人工采纳/拒绝理由和最终验证。
- 与一个成熟实现比较正确性、性能、功能边界和维护成本。

## AI 使用规则

可以让 AI 解释概念、生成测试想法、审查代码和定位错误；不得让 AI 一次性生成最终仓库后直接提交。每关先写自己的接口、假设和验收，再让 AI 参与实现。

## 完成证据

1. 代码仓库与阶段提交。
2. 架构/算法说明。
3. 测试或实验报告。
4. 至少三个失败样本。
5. 成熟实现对照。
6. 10-20 分钟口试记录。
