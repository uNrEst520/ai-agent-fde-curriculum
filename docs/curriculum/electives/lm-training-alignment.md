# XLM · 语言模型训练、对齐与系统优化

- **建议时长：**10 周 · 每周 15-20 小时
- **先修课程：**LM01
- **课程性质：**兴趣选修
- **验收方式：**从零构建 + 方向项目 + 口试

## 方向目标

深入预训练数据、分布式训练、性能优化、扩展规律、指令微调、偏好优化、推理与对齐评测。

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 数据收集、许可、清洗与去重 |
| 2 | tokenizer、数据混合和质量过滤 |
| 3 | 训练计算、优化器、检查点和数值稳定 |
| 4 | GPU 内存、吞吐、混合精度和 profiling |
| 5 | 数据并行、张量/流水并行与通信 |
| 6 | 扩展规律、实验设计和算力预算 |
| 7 | 指令数据、SFT 与评测 |
| 8 | 偏好数据、DPO/RLHF 与奖励建模 |
| 9 | 推理优化、量化、批处理和服务 |
| 10 | 安全对齐、红队和端到端训练报告 |

## 从零构建

[进入逐步构建指南](../../builds/lm-training-alignment.md)

基于自研微型 Transformer 完成数据清洗、训练、SFT 或偏好优化之一，并亲手做性能剖析与瓶颈实验。

## 真实交付

交付一份预算受限的模型训练方案和可重复实验，明确数据来源、算力、收益、风险与停止条件。

## 必交证据

- 可运行代码与可复现实验环境
- 基线、切片指标、失败样本和风险记录
- 面向用户或业务负责人的演示与价值说明

## 反 bypass 口试

1. 数据质量和数据规模冲突时如何设计实验？
2. SFT 与偏好优化分别改变什么？
3. 吞吐提高为什么可能降低有效训练质量？
