# Week 01 必须记住的概念清单

这些不是死记硬背的术语，而是你以后和 Codex 协作、和面试官沟通、和客户交付时必须能说出口的基础语言。

## 1. PowerShell

- `cd`：进入目录。
- `Get-ChildItem`：查看当前目录下的文件和文件夹。
- `Get-Content`：查看文本文件内容。
- 引号路径：路径里有空格或中文时，通常用引号包起来。
- 脚本：一组提前写好的命令，例如 `scripts\start-dev.ps1`。

你不需要背所有命令，但要知道：命令通常由“程序名 + 参数 + 路径/目标”组成。

## 2. 项目目录

- `src`：前端页面和交互。
- `backend/app`：后端 API 和业务逻辑。
- `backend/tests`：测试代码。
- `scripts`：启动、检查、自动化脚本。
- `README.md`：项目说明书。
- `package.json`：前端项目的命令菜单和依赖说明。

## 3. Git/GitHub

- 仓库：一个项目的版本管理空间。
- 变更：文件被新增、修改或删除。
- 提交：把一组变更保存成一个版本。
- 分支：一条独立的开发线。
- README：让别人快速理解项目的入口。

## 4. 前端

- 前端不是只做视觉，它负责接收用户动作。
- `fetch` 是前端请求后端 API 的常见方式。
- 状态可以理解为页面当前记住的数据。
- 组件可以理解为页面里可复用的一块功能。

## 5. API

- API 是前端和后端对话的门。
- `POST` 常用于提交数据或触发任务。
- `GET` 常用于读取结果。
- JSON 是前后端交换结构化数据的常见格式。
- `job_id` 是让系统追踪某一次分析任务的编号。

## 6. FastAPI

- FastAPI 是 Python 里常用的后端 API 框架。
- 路由可以理解为一个接口地址。
- 后端函数接收请求，处理数据，再返回结果。
- 自动文档和类型提示能让 API 更容易调试和交付。

## 7. 测试

- 测试是自动化验收。
- `pytest` 常用于 Python 项目测试。
- 测试能证明某个关键功能仍然存在。
- 测试失败不是坏事，它是在告诉你哪里和预期不一致。

## 8. 训练处方

- 训练处方不是普通建议。
- 它把用户目标、视频证据、动作问题和训练规则组合成下一步行动。
- 它可以被讲成早期的规则 Agent 决策层。
- 后续可以升级为更复杂的 Agent：收集画像、读取历史、判断弱项、选择训练模块、生成计划、追踪反馈。

## 9. 反 bypass

- bypass 是指 AI 做完了，但你不知道它为什么这么做，也不知道怎么验收。
- 反 bypass 不是要求你手写所有代码。
- 反 bypass 是要求你能解释目标、文件位置、数据流、测试命令和验收结果。

## 10. FDE 式交付

- FDE 不是只写功能，而是把技术推进真实业务现场。
- 真实交付要考虑用户流程、组织采用、数据来源、权限、风险、培训和验收。
- AI 从“能用”到“上岗”，中间差的是流程设计和交付能力。
