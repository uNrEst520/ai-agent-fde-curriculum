# SE04 · 计算机系统与网络

- **建议时长：**5 周 · 每周 12-15 小时
- **先修课程：**SE01
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

进程、内存、文件、网络和并发是部署故障、性能问题与安全边界的底层原因。AI 应用工程师不必成为内核专家，但必须能建立可调试的系统模型。

## 学习目标

- 解释程序、进程、线程、端口、文件描述符和环境变量
- 定位 CPU、内存、磁盘、网络与并发瓶颈
- 理解容器隔离、缓存和超时如何影响 AI 服务

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 二进制、内存、调用栈与程序执行 |
| 2 | 进程、线程、并发、锁和死锁 |
| 3 | 文件系统、权限、环境和日志 |
| 4 | TCP/IP、DNS、TLS、端口、超时和重试 |
| 5 | 缓存、队列、容器与系统观察综合实验 |

## 从零构建

实现进程观察器、带过期策略的缓存和一个基于 TCP 的简化请求响应协议。

## 真实交付

为一个现有服务完成容器化、资源限制、健康检查、超时重试和故障排查手册。

## 主要资源

- [Computer Systems: A Programmer's Perspective](https://csapp.cs.cmu.edu/)
- [Beej's Guide to Network Programming](https://beej.us/guide/bgnet/)
- [Docker 官方入门](https://docs.docker.com/get-started/)

资源用于建立主线，最新论文与工具变化从 [研究雷达](../../../research/README.md) 进入候选，不会未经验证直接替换稳定原理。

## 必交证据

- 一次内存或并发故障实验
- 网络请求全链路说明
- 容器与宿主机边界清单

## 反 bypass 口试

1. 程序启动后操作系统为它创建了什么？
2. 超时和重试为什么可能放大故障？
3. 容器解决了什么，没有解决什么？
