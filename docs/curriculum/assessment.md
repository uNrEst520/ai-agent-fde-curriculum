# 评测与毕业标准

## 五层评测

| 层级 | 验证内容 | 典型证据 |
| --- | --- | --- |
| Explain | 能否用自己的话解释 | 口试、概念图、术语对照 |
| Trace | 能否在系统中找到并追踪 | 文件、函数、API、日志和数据流 |
| Operate | 能否运行和诊断 | 启动记录、错误定位、环境说明 |
| Modify | 能否借助 AI 安全修改 | 代码差异、测试、回归和风险说明 |
| Deliver | 能否产生业务结果 | 用户反馈、采用率、质量、成本和复盘 |

只达到 Operate 不等于具备工程交付能力；只达到 Modify 也不等于具备 FDE 能力。

## 单课通过标准

- 总分至少 70/100。
- Explain、构建正确性和评测项不得为零。
- 必须提交可运行代码或等价实验、自动测试和失败证据。
- 无法解释、定位或修改的 AI 生成内容不能计入核心成果。
- 口试失败后可以重做，但必须增加一次新的故障挑战。

## 主线结业标准

1. 完成全部主线课程，或通过对应挑战测试并提交等价构建证据。
2. 完成至少一门兴趣选修。
3. 完成至少一个从零实现的模型或系统，并与成熟实现对照。
4. 为主要 AI 项目建立不少于 50 条、包含边界和对抗案例的评测集。
5. 完成一次故障定位、一次安全红队和一次生产回滚演练。
6. 完成 FD02 企业 AI 上岗毕业项目，并通过演示、口试、用户验收和价值复盘。

## 长期等级评审

L3 及以上采用作品集答辩：

- 项目必须真实运行过，或在高保真环境中持续测试。
- 展示成功案例之外，还要展示失败、事故和修复。
- 说明业务指标、模型指标、系统指标和风险指标。
- 评审者可以要求现场修改一个边界清楚的小功能。
- L4 还需要多角色采用、跨系统接入或重复 FDE 交付证据。
