# 从 AI 使用者到 AI 业务交付者

## 课程定位

这不是一门“AI 工具大全”课，也不是传统程序员训练营，而是一套面向普通人的 AI Builder / FDE 式业务交付课程。目标是让学员从会提问、会使用 Codex/Claude Code，升级为能发现业务问题、设计 Agent 流程、用 AI 工具交付可运行产品，并能向老板、客户或面试官讲清价值的人。

课程的核心角色不是“炫技型 AI 玩家”，而是“前置部署型 AI 业务交付者”：帮助 AI 进入企业，从“能用”走向“上岗”，从“展示能力”走向“交付结果”。

## 目标学员

- 想靠 AI 转行的人。
- 已经会用 AI 工具，但不知道如何变现的人。
- 想在公司内部做 AI 提效的人。
- 有行业经验，想做一人公司或服务创业公司的人。
- 不想和传统程序员硬卷，但愿意补足必要工程理解的人。

## 核心卖点

1. 不卖焦虑，卖可交付能力。
2. 不只教工具，教业务问题识别。
3. 不要求手写全部代码，但要求能解释、验收和接管。
4. 不做玩具项目，围绕真实业务场景做作品集。
5. 每周作业批改，帮助学员从 bypass 变成能交付。

## 正式课程的五大能力模块

1. 发现高价值问题的能力：识别高频、刚需、可量化、可付费的业务流程，而不是为了用 AI 而用 AI。
2. 设计人机协作流程的能力：拆清楚哪些步骤交给模型，哪些步骤由人判断，哪些步骤调用工具或系统执行。
3. AI 输出验收和风控能力：建立测试样例、检查标准、错误兜底、权限边界和人工复核机制。
4. 工程化交付能力：用 Codex、API、前后端、数据库、自动化脚本和测试，把方案做成可运行、可演示、可复盘的产品。
5. 领域知识 + AI 的复合能力：把自己的行业经验、客户场景、业务语言和 AI 能力结合起来，形成普通工具用户难以复制的差异化。

## FDE 启发：从“能用”到“上岗”

FDE（Forward-Deployed Engineer，前置部署工程师）的启发在于：真正有价值的 AI 人才不只是坐在屏幕后面写功能，而是能进入业务现场，理解组织流程、用户阻力、数据限制、风险边界和验收标准。

这门课会加入 FDE 式训练：每个项目都要说明目标用户、原有流程、AI 介入点、人机分工、上线方式、成功指标和风险兜底。学员最终交付的不只是 Demo，而是一份“AI 上岗方案”。

## 12周课程结构

### 阶段一：工程地图

- PowerShell、Git、项目目录。
- Python 基础、JSON、虚拟环境。
- FastAPI 和 HTTP API。
- 前端页面、状态、fetch。

### 阶段二：AI 应用与 Agent 基础

- LLM 输入输出。
- 结构化结果。
- 工具调用。
- OpenAI Agents SDK。
- 简单 Agent 流程。

### 阶段三：可控 Agent

- RAG 与知识库。
- LangGraph 状态流。
- MCP 工具连接。
- Guardrails、tracing、eval。

### 阶段四：商业交付

- 真实项目作品集。
- 客户/老板提案。
- 成本、风险、边界。
- 面试故事和 Demo。
- FDE 式部署方案：让 AI 进入真实工作流并被团队持续使用。

## 学完成果

- 一个可展示的 AI 业务交付项目。
- 一套 Agent 工作流设计能力。
- 一份作品集与项目说明。
- 一份 AI 上岗方案：包含业务流程、Agent 设计、验收标准、风控边界和部署计划。
- 一套能向创始人/客户/面试官表达的商业话术。
- 一种使用 Codex 但不完全 bypass 的开发方法。
