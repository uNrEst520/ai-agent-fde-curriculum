# 能力等级模型

## 八个能力域

课程以八个能力域评估成长，不用“看完多少视频”代替能力。

| ID | 能力域 | 核心问题 |
| --- | --- | --- |
| BIZ | 高价值问题发现 | 这个问题值得用 AI 解决吗？ |
| HAI | 人机协作流程 | 哪些步骤给模型，哪些给规则、工具和人？ |
| ENG | 工程化交付 | 系统能否运行、测试、部署、监控和维护？ |
| EVAL | 验收与风控 | 如何证明结果正确、可靠、安全且可追责？ |
| DOMAIN | 领域知识复合 | 行业知识如何进入数据、规则、评测和体验？ |
| CS | 计算与软件基础 | 能否理解代码、API、数据、状态、存储和系统边界？ |
| AGENT | Agent 系统设计 | 能否设计上下文、工具、状态、计划和恢复机制？ |
| FDE | 前置部署与采用 | 能否进入现场、接入流程、培训用户并量化价值？ |

## 五个等级

### L1：观察与解释

- 能运行现有系统。
- 能说清用户流程和主要模块。
- 能在指导下找到接口、日志和测试。
- 仍然依赖明确步骤完成任务。

### L2：AI 辅助独立交付

- 能把清楚的需求拆成页面、接口、数据和验收标准。
- 能让 Coding Agent 实现功能，并自行审查和测试。
- 能建立小型评测集，识别常见失败。
- 能向非技术人员讲清价值、限制和风险。

### L3：生产级负责人

- 能处理权限、异常、并发、成本、监控和版本回归。
- 能设计 RAG、工具调用或工作流，并用数据比较方案。
- 能负责上线、运行和迭代，而不仅是完成 Demo。
- 能对业务指标和技术指标同时负责。

### L4：FDE / 解决方案负责人

- 能在模糊业务现场发现高价值问题。
- 能跨数据、流程、系统、人员和治理边界设计方案。
- 能推动试点、培训、采用和组织变革。
- 能建立可复制的行业解决方案。

### L5：方法与平台领导者

- 能定义组织级 AI 架构、评测和风险标准。
- 能领导多项目和跨职能团队。
- 能把项目经验沉淀为平台、方法论和人才培养体系。
- 能判断新研究何时值得进入生产体系。

## 评定规则

等级不是八项能力的平均值。申请某一级别时：

- 八项能力至少六项达到该级别。
- EVAL、ENG、BIZ 三项不得低于目标级别。
- 必须有真实项目证据，不接受只靠选择题或证书认定。
- L3 以上必须提供运行期数据、事故或失败复盘和用户采用证据。

## 当前阶段目标

12 周共同核心目标是从 L0 / L1 进入 L2。之后每个专修和 FDE 项目提升一个或多个能力域，通常需要 6-18 个月形成稳定的 L3 能力。
