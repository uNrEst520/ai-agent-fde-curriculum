# AP02 · RAG 与知识工程

- **建议时长：**5 周 · 每周 12-15 小时
- **先修课程：**AP01、DE01
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

企业 AI 的关键常常不是模型知道多少，而是能否使用正确、最新、有权限且可引用的组织知识。

## 学习目标

- 拆解文档接入、解析、切分、检索、重排和生成链路
- 建立检索与答案两个层级的评测
- 处理更新、冲突、权限、引用和无答案场景

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 知识边界、文档解析、元数据和切分 |
| 2 | 倒排检索、TF-IDF、embedding 与相似度 |
| 3 | 混合检索、重排、查询改写和上下文组装 |
| 4 | 引用、权限、更新、冲突与提示注入 |
| 5 | 离线评测、失败分析与企业知识应用 |

## 从零构建

[进入逐步构建指南](../../builds/rag-engine.md)

从零实现文本切分、倒排索引、TF-IDF/向量相似度、top-k 检索、引用拼装和小型评测器。

## 真实交付

使用成熟存储构建有权限过滤、增量更新、来源引用和离线回归集的知识系统。

## 主要资源

- [Retrieval-Augmented Generation 原始论文](https://arxiv.org/abs/2005.11401)
- [Hugging Face NLP Course](https://huggingface.co/learn/nlp-course/)
- [BEIR Benchmark](https://github.com/beir-cellar/beir)

资源用于建立主线，最新论文与工具变化从 [研究雷达](../../../research/README.md) 进入候选，不会未经验证直接替换稳定原理。

## 必交证据

- 检索命中与答案正确分层报告
- 至少 30 条评测集
- 过期、冲突、越权和提示注入案例

## 反 bypass 口试

1. 检索正确但答案错误可能发生在哪里？
2. 为什么切分不是越小越好？
3. 系统应如何回答“没有足够证据”？
