# AP03 · 工具调用、MCP 与权限边界

- **建议时长：**5 周 · 每周 12-15 小时
- **先修课程：**AP01、SE04
- **课程性质：**主线课
- **验收方式：**构建项目 + 交付任务 + 口试

## 为什么学

当模型可以读取数据和执行动作时，能力与风险同时扩大。工程师需要理解协议生命周期、参数验证、授权、同意和审计。

## 学习目标

- 设计明确、可验证、可取消的工具接口
- 实现 MCP 服务端与客户端的核心交互
- 建立最小权限、人工确认和工具威胁模型

## 周计划

| 周 | 主题与实践 |
| --- | --- |
| 1 | 函数调用、schema、参数验证和幂等 |
| 2 | 工具选择、执行结果、错误与补偿 |
| 3 | MCP 架构、生命周期、工具、资源和提示 |
| 4 | 传输、授权、用户同意、审计与恶意内容 |
| 5 | 自建服务、客户端集成和安全测试 |

## 从零构建

先设计一个简化工具协议和执行器，再实现最小 MCP 服务端与客户端，记录完整请求、结果和错误轨迹。

## 真实交付

连接一个真实但低风险的业务工具，加入权限范围、人工确认、速率限制、审计和撤销机制。

## 必交证据

- 协议时序图
- 权限矩阵和威胁模型
- 越权、重复执行和恶意参数测试

## 反 bypass 口试

1. 工具描述为什么也是攻击面？
2. 读取权限与执行权限应如何分离？
3. 幂等和补偿各解决什么问题？
