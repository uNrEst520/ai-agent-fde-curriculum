# 学习与反 bypass 契约

## 可以交给 AI 的工作

- 生成初稿、代码和测试。
- 搜索代码库、解释函数和定位错误。
- 执行重复操作、格式化和整理资料。
- 提出方案、反例和检查清单。

## 学习者必须负责的工作

- 明确业务问题和成功标准。
- 判断 AI 的方案是否适合当前场景。
- 说清输入、处理、输出、存储和权限。
- 检查代码改动范围、异常、测试和安全。
- 运行验证并保留证据。
- 对最终交付和用户影响负责。

## 每次 AI 辅助开发后的六问

1. 这个改动解决什么用户问题？
2. 数据从哪里来，到哪里去？
3. 哪三个文件或函数最关键？
4. 最可能出现哪三个失败？
5. 什么证据证明它现在可用？
6. 如果上线出错，如何发现、停止和恢复？

## 口试规则

每周作业提交后进行 10-20 分钟口试。导师从以下类型随机追问：

- 换一种输入会怎样？
- 删除某个模块会怎样？
- 为什么使用 LLM 而不是规则？
- 测试保护了哪项行为？
- 这个结论的证据在哪里？
- 真实用户不按预期操作怎么办？

回答不要求背代码，但必须能定位、推理和验证。
