# 公开仓库隐私与发布安全

## 基本原则

公开仓库中的文件、图片、日志和 Git 历史都可能被搜索、复制和长期缓存。发布前不仅要检查代码，还要检查文档、截图、生成报告和已经提交的历史。

## 不得公开

- 本机用户名、用户目录、桌面和下载目录等绝对路径。
- 个人邮箱、电话、地址、设备名和账号标识。
- API 密钥、访问令牌、Cookie、密码、私钥和真实环境变量。
- 客户名称、未公开项目、合同、内部流程和可识别业务数据。
- 带有终端用户名、浏览器账号、文件路径或后台数据的截图。
- 原始视频、照片和文档中的身份信息及文件元数据。

## 统一占位符

| 信息 | 占位符 |
| --- | --- |
| 课程仓库根目录 | `<CURRICULUM_ROOT>` |
| 实践项目根目录 | `<PROJECT_ROOT>` |
| 用户目录 | `<USER_HOME>` |
| API 密钥 | `<API_KEY>` |
| 客户或组织 | `<ORGANIZATION>` |
| 示例用户 | `<EXAMPLE_USER>` |

教学命令使用占位符时，应先说明学习者需要把它替换成自己的路径，不应把维护者的真实目录作为示例。

## 提交前检查

```powershell
python scripts\validate_curriculum.py
git diff --cached --check
git diff --cached --name-only
```

还要人工检查新增图片和视频。文本扫描无法发现截图中的用户名、文件路径、浏览器头像和业务数据。

## 发现泄露后的处理

1. 立即停止继续传播。
2. 密钥或令牌必须先撤销和轮换，删除文件并不等于密钥安全。
3. 修复当前文件。
4. 重写包含敏感信息的 Git 历史，并安全强制推送。
5. 检查 Release、Actions Artifact、Fork、缓存和其他镜像。
6. 记录事件原因，并补充自动检查规则。

## 自动检查边界

本仓库的验证脚本会拦截常见绝对路径、用户目录、个人邮箱和密钥格式，但它不能证明仓库绝对安全。新类型数据、图片内容、隐写信息和经过编码的秘密仍需人工审查。
